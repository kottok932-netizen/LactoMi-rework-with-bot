# LactoMi: готовый бесплатный MVP

Я уже собрал проект так, чтобы его можно было запустить **без обязательных платежей** на старте:

- статический сайт на **Cloudflare Pages**;
- серверная функция на **Pages Functions**;
- AI через **Cloudflare Workers AI**;
- чтение PDF через **Workers AI Markdown Conversion**;
- защита формы через **Cloudflare Turnstile**;
- базовая защита от абьюза через **Cloudflare WAF rate limiting**.

## Что уже сделано

### 1. Полностью пересобрана главная страница
Сайт теперь заточен не под презентацию проекта, а под пользователя:

- большой акцент на загрузку PDF;
- блок «как это работает»;
- короткие карточки по ключевым микроорганизмам;
- аккуратный блок о LactoMi;
- FAQ и дисклеймеры;
- отдельная страница по приватности.

### 2. Добавлен реальный серверный маршрут `/api/analyze`
Он умеет:

- принимать PDF;
- проверять Turnstile;
- конвертировать PDF в Markdown через Cloudflare Workers AI;
- отправлять извлечённый текст в LLM;
- возвращать пользователю готовую расшифровку.

### 3. Проект сделан без обязательных внешних AI-ключей
В этой сборке не нужен OpenAI API key и не нужен Gemini API key.
Всё завязано на ваш бесплатный Cloudflare-аккаунт и AI binding.

### 4. Сделан упор на понятный ответ
Бот не должен копировать чужой кейс. Он:

- берёт реальные данные из загруженного PDF;
- сравнивает их с референсами из самого документа;
- отвечает по структуре: итог, норма, отклонения, простое объяснение, что обсудить с врачом, мягкое упоминание LactoMi, дисклеймер.

# Что вам нужно сделать самому

## Шаг 1. Загрузить проект в GitHub

1. Создайте новый репозиторий на GitHub.
2. Загрузите туда все файлы из этой папки.

Если хотите через терминал:

```bash
git init
git add .
git commit -m "LactoMi MVP"
git branch -M main
git remote add origin https://github.com/ВАШ-ЛОГИН/ВАШ-РЕПО.git
git push -u origin main
```

---

## Шаг 2. Создать бесплатный Cloudflare Pages проект

1. Откройте Cloudflare.
2. Перейдите в **Workers & Pages**.
3. Нажмите **Create application**.
4. Выберите **Pages**.
5. Подключите GitHub-репозиторий.
6. Build command оставьте пустой.
7. Build output directory: `.`
8. Нажмите Deploy.

> Если удобнее через консоль, можно использовать `wrangler pages deploy .`, но для первого запуска GitHub-интеграция проще.

---

## Шаг 3. Подключить Workers AI binding

После первого деплоя:

1. Откройте ваш Pages-проект.
2. Зайдите в **Settings**.
3. Откройте **Bindings**.
4. Нажмите **Add binding**.
5. Выберите **Workers AI**.
6. Название binding: `AI`
7. Сохраните.
8. Перезапустите деплой.

Это главный шаг, без него `/api/analyze` не заработает.

---

## Шаг 4. Создать Turnstile

1. В Cloudflare откройте **Turnstile**.
2. Создайте новый widget для вашего домена.
3. Получите:
   - `sitekey`
   - `secret`

### Что сделать дальше

#### 4.1. Вставить sitekey в фронтенд
Откройте `assets/config.js` и найдите:

```js
window.LACTOMI_CONFIG = {
  turnstileSiteKey: "PASTE_YOUR_TURNSTILE_SITE_KEY_HERE"
};
```

Замените значение на ваш реальный `sitekey`.

#### 4.2. Добавить secret в Cloudflare Pages
Откройте ваш Pages-проект:

**Settings -> Variables and Secrets**

Добавьте secret:

- `TURNSTILE_SECRET` = ваш secret key

---

## Шаг 5. Добавить необязательные переменные

В **Settings -> Variables and Secrets** можно добавить ещё:

- `AI_MODEL` = `@cf/qwen/qwen3-30b-a3b-fp8`
- `PRODUCT_NAME` = `LactoMi`

Можно не добавлять: тогда проект использует эти значения по умолчанию.

---

## Шаг 6. Настроить базовую защиту от абьюза

### 6.1. DDoS
Ничего дополнительно включать не нужно: базовая Cloudflare-защита уже работает на уровне платформы.

### 6.2. Rate limiting rule для `/api/analyze`
На бесплатном плане Cloudflare вам хватит одного правила именно для этого маршрута.

Создайте правило для пути:

```text
/api/analyze
```

Рекомендуемый старт:

- 10 запросов за 1 минуту на IP
- действие: `Managed Challenge` или `Block`

Если хотите мягче:

- 5 запросов за 30 секунд
- действие: `Managed Challenge`

Проверьте:

3. Уходит ли PDF на `/api/analyze`.

# Локальный запуск

## Установка

```bash
npm install
```

## Локальная разработка

```bash
npm run dev
```

Это запустит Pages локально и подключит AI binding через Wrangler.

### Локальные переменные
Скопируйте `.dev.vars.example` в `.dev.vars`:

```bash
cp .dev.vars.example .dev.vars
```

В `.dev.vars` укажите хотя бы:

```env
TURNSTILE_SECRET=""
PRODUCT_NAME="LactoMi"
AI_MODEL="@cf/qwen/qwen3-30b-a3b-fp8"
```

Если Turnstile ещё не настроен, `TURNSTILE_SECRET` можно временно оставить пустым для локального теста.

# Как сейчас работает бот

1. Пользователь загружает PDF.
2. Pages Function принимает файл.
3. Turnstile подтверждает, что это не бот.
4. `env.AI.toMarkdown()` извлекает текст из PDF.
5. Модель читает извлечённый текст и отвечает по заданной структуре.
6. Пользователь видит понятную расшифровку.

---

# Важное ограничение этого бесплатного MVP

Этот вариант лучше всего работает с **цифровыми PDF**, где есть нормальный текстовый слой.

Если анализы будут приходить как:

- сканы;
- фото;
- плохие распечатки;
- PDF, где текст лежит только картинкой,

то качество извлечения может падать.

Для такого случая следующим этапом можно сделать более дорогой режим:

- raw PDF vision-анализ;
- OCR fallback;
- отдельные шаблоны под разные лаборатории.

Но для старта, чтобы запуститься без бюджета, текущей версии достаточно.
